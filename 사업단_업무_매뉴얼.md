<img src="https://r2cdn.perplexity.ai/pplx-full-logo-primary-dark%402x.png" style="height:64px;margin-right:32px"/> # 사업단 업무 매뉴얼을 튜토리얼로 웹 프로그램을 짜서 업무인수 인계시 원할하게 하고자 합니다.핵심적인 사업비 수입과 지출처리 시스템과 화면 단위 업무 처리 절차를 최대한 세부적으로 시뮬레이션할수 있는 프로그램을짜고자 합니다. 업무 절차별 해당 시스템을 언급해주고, 관련 화면을 참조할수 있게 해주어야 합니다. 시스템은 은행 이체 처리 ' e-Branch' '연구비종합관리 시스템', '베스트케어' 시스템 , 'Ezbaro '시스템 4가지가 주요 시스템이며 고유의 기능과 업무 처리 흐름을 명확히 설명해주어야 함. 연구비 종합시스템은 청구(결의)서라고 하여 처리하고, 이후 베스트케어에서 '지출 결의서' 처리가한번더있는구조이다. 크게 수입 지출 업무를 처리하는 구조로 나뉘고 , 각각의 지출 성격에 따를 분류로 처리 흐름과 증빙, 주의사항 강조하여 작성해달라. 요청하신 내용을 바탕으로, “사업단 수입·지출 업무 인수인계를 위한 웹 튜토리얼 프로그램”의 전체 설계 틀을 먼저 잡아보겠습니다.[^1] (※ 아래는 화면 설계·흐름·주의사항 중심의 초안입니다. 이후 실제 화면 캡처를 하나씩 붙여 확장하면 됩니다.) *** ## 1. 전체 구조(메뉴 맵) 제안 웹 튜토리얼 메인 메뉴 예시: 1. 대시보드 - 오늘 처리해야 할 결의/지출 건, 미완료 단계 표시. 2. 업무 개요 - 사업단 재정 구조, 4대 시스템(e-Branch, 연구비종합관리, 베스트케어, Ezbaro) 역할 요약.[^1] 3. 수입 업무 튜토리얼 - 수입 유형별(정부·지자체, 산학협력단/병원 내부이체, 기타) 처리 절차. 4. 지출 업무 튜토리얼 - 인건비, 물품구입, 용역·행사비, 출장비 등 지출 성격별 절차 및 증빙.[^1] 5. 시스템별 상세 튜토리얼 - e-Branch - 연구비종합관리시스템(청구/결의서) - 베스트케어(지출결의서) - Ezbaro (또는 RCMS·GAIA 등 연구비 연계 시스템 설명 포함).[^1] 6. 체크리스트 & 자주 하는 실수 - 단계별 체크리스트, 잦은 오류 사례 정리.[^1] 7. 계정·문의처 - 각 시스템 담당자 연락처, ID/PW 신청 방법.[^1] 각 메뉴는 “업무 흐름도 + 단계별 화면 시뮬레이션(이미지 또는 더미 화면) + 체크리스트” 구조로 통일하면 좋습니다. *** ## 2. 4대 시스템 역할 정리 | 시스템명 | 주요 기능/역할 | 연계 위치(업무 흐름 내) | | :-- | :-- | :-- | | e-Branch | 은행 계좌 이체, 수입금 입금 확인, 계좌 간 이체 처리.[^1] | 수입금 입금·이체, 지출 단계의 실제 송금 처리. | | 연구비종합관리 시스템 | 과제/사업별 예산편성, 집행 관리, 청구(결의)서 생성.[^1] | 지출의 1차 결의(청구) 단계, 예산 잔액·항목 확인. | | 베스트케어 | 병원 내부 회계·지출결의 처리, 세목 코드, 전표 생성.[^1] | 연구비종합관리에서 올라온 결의를 실제 지출결의로 확정. | | Ezbaro | 정부·연구재단 등 대외 연구비 집행 보고, 세부 집행 내역 업로드.[^1] | 지출 후 대외보고 단계, 집행 실적 등록·정산 보고. | 튜토리얼에서는 각 시스템마다 “로그인 → 메뉴 위치 → 입력 필드 설명 → 저장·승인 흐름”을 실제 화면 캡처 기반으로 한 단계씩 보여주도록 설계합니다. *** ## 3. 수입·지출 전체 업무 흐름(상위) ### 3.1 공통 기본 흐름 1. 예산 편성/배정 2. 수입 발생·입금(e-Branch 확인, 내부 이체 등).[^1] 3. 지출 계획 수립 및 집행(연구비종합관리 → 베스트케어 → e-Branch 송금).[^1] 4. 사후 정산(Ezbaro, RCMS 등 대외보고).[^1] 웹 튜토리얼에서는 각 단계별로 어떤 시스템을 열어야 하는지, 어떤 화면을 봐야 하는지 명시합니다. *** ## 4. 수입 업무 튜토리얼 설계 ### 4.1 수입 유형 분류 - 정부·연구재단·지자체 연구비 유입 - 타 기관 수탁·용역 대금 - 병원/학교 내부 재원(전입금 등) - 기타 후원·기부금 등 각 유형별로 “입금 경로와 입금 확인 방법, 연구비시스템에 세팅되는 방식”이 다르므로, 사용자에게 아래와 같이 안내합니다.[^1] ### 4.2 수입 업무 화면 흐름 예시 1. e-Branch: 입금 확인 - 화면: “입금내역 조회” - 기능: 날짜·계좌 선택 → 입금자명·금액 확인. - 주의: 입금자명/프로젝트명 불일치, 금액 오입력 주의.[^1] 2. 연구비종합관리: 수입 예산 반영 - 화면: 과제 선택 → 예산 내역/수입 등록 화면. - 기능: 회계 코드, 사업코드, 수입 항목 선택 후 금액 입력. - 주의: 사업(과제)번호, 계정과목, VAT 포함 여부 확인.[^1] 3. 베스트케어(필요 시): 내부 회계 반영 - 화면: 수입 전표(또는 해당 메뉴) 등록. - 기능: 수입 계정·대체 계정 선택, 금액 입력. - 주의: 수입 계정 코드 오류 시 정산 시점에 문제 발생.[^1] 튜토리얼 페이지에는 각 단계마다 “1단계: e-Branch 입금조회 (화면 캡처)” “2단계: 연구비종합관리 수입등록 (화면 캡처)” 형태로 이미지/더미 UI를 배치합니다. *** ## 5. 지출 업무 튜토리얼 설계(핵심) 연구비 지출은 1) 연구비종합관리 시스템에서 ‘청구(결의)서’ 생성 2) 베스트케어에서 ‘지출결의서’ 한 번 더 처리 3) e-Branch에서 실제 이체 4) Ezbaro 등 대외시스템 보고 의 구조로 구성됩니다.[^1] ### 5.1 지출 성격별 분류 튜토리얼에서 최소 다음과 같이 분류해 각 메뉴를 따로 구성하는 것이 좋습니다.[^1] - 인건비·학생인건비·강사료 - 물품·소모품 구입 - 연구·용역비(통계·디자인·분석, 기술료 등) - 출장비(교통, 숙박, 일비 등) - 회의비·식비·행사비 - 학회·교육비, 등록비 - 소프트웨어·서비스 구독료(SaaS) - 장비·시설 사용료(Core Facility 등).[^1] 각 메뉴마다 “필수 증빙 / 연구비종합관리 입력 요령 / 베스트케어 지출결의 작성 / e-Branch 송금 / Ezbaro 보고 포인트 / 주의사항” 을 동일 포맷으로 보여줍니다.[^1] *** ## 6. 지출 업무 단계별 화면 흐름(예시) ### 6.1 연구비종합관리 시스템 – 청구(결의)서 1. 과제(사업) 선택 - 화면: 과제 리스트 → 해당 과제 클릭.[^1] - 체크: 과제 기간, 집행기간, 잔액 확인. 2. 청구서(결의서) 작성 - 화면: “지출청구/결의” 메뉴. - 입력 항목: - 지출 구분(인건비/물품/용역/출장비 등) - 공급업체/수령인 정보 - 계정과목·세목 코드 - 금액(공급가, 부가세 구분) - 집행일자, 적요(사용 목적).[^1] - 주의: - 예산 항목과 지출 항목이 일치해야 하며, 다른 항목으로 집행 시 사전 변경 필요.[^1] - 과제 기간 밖 집행 금지(기간 체크).[^1] 3. 첨부파일 업로드 - 견적서, 계약서, 세금계산서/영수증, 카드 전표, 소명자료 등.[^1] - 주의: - 금액·공급자 정보·날짜가 시스템 입력값과 일치하도록 확인. - 전자세금계산서 XML 파일 요구 여부(파일에서 언급).[^1] 4. 결재 상신 - 연속 결재선 설정 후 상신. - 상태값: 작성 → 검토 → 승인.[^1] 튜토리얼 화면 예시: - 좌측에 실제 화면 캡처 또는 더미 UI - 우측에 “필드별 설명 + 체크리스트 + 자주하는 오류” 영역. *** ### 6.2 베스트케어 – 지출 결의서 연구비종합관리에서 승인된 청구서 기반으로 베스트케어에서 지출결의서를 한 번 더 작성/확정합니다.[^1] 1. 지출결의 메뉴 진입 - 화면: “지출결의 등록/수정”. - 기능: 사업코드·과제코드·계정코드 선택.[^1] 2. 연구비시스템과 금액·항목 일치 여부 확인 - 공급가/부가세, 총액.[^1] - 지출일자, 거래처 정보. 3. 증빙번호/전표번호 입력 - 전자세금계산서 번호, 카드 승인번호 등.[^1] 4. 결의서 저장 및 결재 - 회계·경영지원 부서 결재 선.[^1] 주의사항: - 연구비종합관리 시스템의 결의서와 베스트케어 전표가 1:1 매칭되도록 관리. - 계정코드/세목을 베스트케어 기준으로 재검토(병원 회계 기준).[^1] *** ### 6.3 e-Branch – 실제 이체 처리 베스트케어에서 지출결의가 승인된 이후 실제 송금은 e-Branch에서 이루어집니다.[^1] 1. 로그인 - ID/PW는 사업단 전용 계정, 매뉴얼에 ID 발급 절차와 담당자 연락처 포함.[^1] 2. 이체 메뉴 선택 - “타기관 이체”, “대량이체(급여성)”, “지로/공과금” 등 지출 성격에 맞는 메뉴 선택.[^1] 3. 수취인 정보 입력 - 계좌번호·예금주·은행명. - 금액, 이체일자, 적요.[^1] 4. 이체 승인 - OTP/보안카드, 이중 승인 프로세스 안내.[^1] 주의사항: - 연구비·베스트케어 금액과 동일해야 하며, 수취인 계좌 확인 필수. - 인건비·학생인건비 등 대량 지급 시 엑셀 업로드 템플릿 사용법 안내.[^1] *** ### 6.4 Ezbaro – 대외보고/정산 지출 후 국가연구비 등은 Ezbaro(또는 RCMS·GAIA 등)에서 집행 내역을 등록·보고해야 합니다.[^1] 1. 과제 선택 - 과제번호, 연구책임자명으로 검색.[^1] 2. 지출내역 등록 - 지출일자, 집행금액, 집행항목(인건비/연구활동비 등) 입력. - 증빙번호, 집행기관명.[^1] 3. 증빙 첨부 - 세금계산서, 영수증, 인건비 지급대장 등.[^1] 4. 저장 및 제출 - 분기/연도별 집행실적 보고.[^1] 주의사항: - 내부시스템(연구비·베스트케어)과 Ezbaro 금액·항목 일치 여부 상시 체크. - 간접비/직접비 구분, 항목별 상한·제한 규정 반영.[^1] *** ## 7. 튜토리얼 화면 구성 규칙(실제 웹 프로그램 디자인 가이드) 각 “업무 시나리오” 페이지는 다음 블록으로 구성하면 좋습니다. 1. 시나리오 개요 - 예: “국가연구비 과제에서 통계분석 용역비 300만원 지출”. 2. 업무 흐름 다이어그램 - 연구비종합관리(청구서) → 베스트케어(지출결의서) → e-Branch(송금) → Ezbaro(보고).[^1] 3. 단계별 화면 시뮬레이션 - Step 1: 연구비종합관리 – 청구서 작성 (화면 이미지 + 설명) - Step 2: 베스트케어 – 지출결의서 작성 - Step 3: e-Branch – 이체 -
